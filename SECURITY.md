# Security Policy

## Supported Versions

The repository is research software. Security fixes are applied on a
best-effort basis to the latest main branch.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

Instead:
- Open a private GitHub security advisory, or
- Contact maintainers directly if private advisory is not available.

Include:
- impacted file(s) and function(s)
- reproduction steps
- expected risk/impact
- optional patch suggestion

We will acknowledge reports as quickly as possible and coordinate remediation.
